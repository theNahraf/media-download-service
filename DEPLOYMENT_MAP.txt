│                                                                             │
│  "I built a scalable media download service using:"                       │
│                                                                             │
│  Architecture:                                                            │
│  • FastAPI for async HTTP API (handles millions of requests)              │
│  • Celery + Redis for job queue (decoupled processing)                    │
│  • PostgreSQL for data persistence                                        │
│  • S3/MinIO for file storage                                              │
│                                                                             │
│  Why This Design:                                                         │
│  • Async API = high concurrency, no blocking                              │
│  • Celery = horizontal scaling (add workers as needed)                    │
│  • Redis queue = deduplication + progress tracking                        │
│  • Separate storage = stateless servers                                   │
│                                                                             │
│  Deployment:                                                              │
│  • Currently running on Railway (free tier)                               │
│  • Same code works on AWS ECS/Fargate for production                      │
│  • Fully containerized with Docker                                        │
│  • Infrastructure as code (docker-compose.yml)                            │
│                                                                             │
│  Scalability:                                                             │
│  • API can handle 1000s of concurrent users                               │
│  • Workers are stateless, can scale horizontally                          │
│  • Database connection pooling via SQLAlchemy                             │
│  • Redis caching prevents duplicate downloads                             │
